Overview

One of the ways in which X differentaites from other frameworks, is that it's very basic when it comes to styling, and has been built to make it very easy for anyone to customise how it looks. Unlike say, Bootstrap, elements come with minimal styling, so you don't have to waste time overriding styles, and instead can add as you go. The little styling that is included, works by referencing various settings that can be found in the _theme.sass file.

When you download the source files, you'll find the following structure: 

|--- mixins
|--- components
|--- helpers
|_base.sass
| _theme.sass
| styles.sass

Each file in the above folders hich then get imported into the _base.sass file. This file, and the theme file then getsimported into a global styles.sass which then gets compiled into the CSS.

Please note that X uses an autoprefixer - if you are using this theme, you'll need to make sure you add this to your built tool to ensure cross-browser compatiblity.

The Theme File

Included in the download is a boilerplate theme.sass file that contains all the system-wide settings you can change.

Sizing

  • size-border-radius: used on elements with rounded corners, such as buttons, forms, dropdowns etc. If you don't want rounded corners, simply set this to 0.
  • size-base-font: base font size, set on the HTML & Body elements. Also used as the base unit to calculate unit layout helpers.
  • size-container-padding: padding between the content and browser edge.
  • size-grid-gutter: amount of spacing between standard grid columns.
  • size-base-font: max-width set on the .container-wide container.
  • mobile-first: changes the order of the media query, so mobile styles come first. This is a preference for some people, and basically means all mobile styles are declared first, and are then followed by tablet and desktop styles. 

$theme: (
    // Sizing
    size-border-radius: 3px,
    size-base-font: 16px,
    size-container-padding: 20px,
    size-base-spacing: 10px,
    size-grid-gutter: $size-basespacing, 
    size-site-width: 1000px
     
    // Theming
     
    font-family-sans: Helvetica,
     
    // Responsive
     
    mobile-first: true, 
     
    // Imports
     
    import-components: true,
    import-helpers: true,
     
    // Colors
     
    color-brand-primary: null, 
    color-brand-secondary: null, 
    color-keyline: 
    color-notifications:
    shading:
    drop-shadow:
    drop-shaodw-large:
);

Layout Units

Enso using a system of layout units to keep spacing like margins and padding consistent throughout the project.

Helper classes that allow you to add or override an objects margin and padding.

This can be useful, if say, you want to add more margin to a heading elemtn in your interface, or inteed would rather avoid.

In our layout, we set a base unit size (by default, 16px) which is then used to caclulate variants.

  • sizes-quarter-unit: 16px / 4
  • size-half-unit: 16px / 2
  • size-base-unit: 16px
  • size-double-half: 24px
  • size-double-unit: 32px
  • size-triple-unit: 48px
  • size-quadriple-unit: 64px

Responsive Typograhpy

he typgraophy is set using a module typescale, specified in config/text-sizes.scss. 8 sizes are specified:

The mixin takes the various breakpoints specified in the breakpoints config file, and will adapt the text size for each device at that breakpoint.

The different elements come prepacked with theses sizes set. Doing things this way allows ut to get better control over how different elements render on

different devices (I used to use REM units to do this, but find this solution gives more control).

From a designer's perspective, typography is one of the most critial asepcts of any prokect. Managing typgoraphy well, especially with varying viewports makes it even harder. It works in the following way:

  • X includes 7 different standard sizes - 3 sizes above the base font size, and 3 sizes above.
  • The size and line-height is then specified for each type of device, found int he breakpoitns map.
  • To use a size on a new element, use the mixin @responsive-text-size(size).
  • You can speficiy additional breakpoints and sizes in the map. 
$text-sizing: (
  x-small: (
    device-small: (
      font-size: rem(14px)
     ),
     device-medium: (
       font-size: rem(12px)
      ), 
     device-large: (
       font-size: rem(12px)
     )
   ),
  medium: (
   device-small: (
     font-size: rem(16px)
    ),
   device-medium: (
     font-size: rem(18px)
    ), 
    device-large: (
      font-size: rem(18px)
     )
   ),
);

Stacking

Managing z-index order can be a pain - SASS docs makes it easy. In config/z-index is a sass map with various objects and their z-index value.

You can add new objects here, then add to the object using the layers($object) function.

$layers: (
  primary-header-site: 1,
  modal: 2,
  dropdown-menu: 3,
  ...
);
 
 
 
// Use the function
 
.primary-header-site {
  z-index: layers(primary-header-site);
}
